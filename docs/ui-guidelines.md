# 前端 UI 与交互规范

> UI 组件库选型：Naive UI。目标风格：简约、现代、柔和、心理健康场景友好，并尽量避免与常见 Element Plus 后台系统撞车。

---

## 1. 设计原则

1. **简约现代**：页面以卡片、留白、轻阴影为主，减少粗重边框。
2. **心理健康场景友好**：色彩柔和，避免大面积高饱和红色或压迫感强的视觉元素。
3. **信息层级清晰**：状态、风险等级、预约时间、操作按钮应一眼可识别。
4. **操作安全**：驳回、取消、删除、提交结案等关键操作必须二次确认。
5. **可演示性强**：页面应适合答辩演示，核心流程入口清晰，数据状态醒目。

---

## 2. 色彩规范

建议主题色：蓝绿色。

| 用途 | 颜色建议 | 说明 |
|:--:|:--:|:--|
| 主色 | `#18A058` 或偏蓝绿自定义色 | 主要按钮、选中菜单 |
| 辅助蓝 | `#2080F0` | 信息、待审核状态 |
| 背景灰 | `#F5F7FA` | 主内容区背景 |
| 卡片背景 | `#FFFFFF` | 表单和表格卡片 |
| 成功 | `#18A058` | 审核通过、完成 |
| 警告 | `#F0A020` | 排队中、中风险 |
| 危险 | `#D03050` | 高风险、驳回、错误 |
| 弱文本 | `#7A7A7A` | 说明文字 |

风险颜色：

| 风险等级 | 标签颜色 | 行背景建议 |
|:--:|:--:|:--:|
| LOW | success | 默认 |
| MEDIUM | warning | 默认或浅黄 |
| HIGH | error | `#FFF3F3` |
| URGENT | error | `#FFECEC` + 加粗标签 |

## 3. 布局规范

### 3.1 主布局

主布局采用侧边栏 + 顶栏 + 内容区。

- 侧边栏宽度：220px。
- 顶栏高度：56px。
- 内容区 padding：24px。
- 页面卡片间距：16px。
- 表格卡片使用 `n-card`。

### 3.2 页面头部

每个业务页面使用 `PageHeader`：

- 标题：当前页面名称。
- 描述：一句话说明页面用途。
- 右侧：主要操作按钮，例如“新增”“导出”。

示例：

```text
初访预约审核
查看学生提交的初访预约申请，优先处理高风险学生。
[导出] [刷新]
```

### 3.3 搜索区

搜索区使用 `SearchPanel`：

- 放在表格上方。
- 一行最多 4 个搜索项。
- 搜索按钮使用主色，重置按钮使用次要按钮。
- 日期范围统一使用 `n-date-picker type=daterange`。

## 4. 组件使用规范

### 4.1 按钮

| 场景 | 组件 | 类型 |
|:--:|:--:|:--|
| 新增/保存/提交 | `n-button` | primary |
| 搜索 | `n-button` | primary |
| 重置/取消 | `n-button` | default |
| 驳回/删除/释放 | `n-button` | error |
| 查看详情 | `n-button text` | info/default |

同一行操作按钮建议使用文字按钮，避免表格拥挤。

### 4.2 表格

Naive UI `n-data-table` 统一配置：

- `remote=true`：后端分页。
- `loading`：接口加载状态。
- `pagination`：分页对象。
- `row-key`：使用 `id`。
- 长文本使用 `ellipsis` 或 tooltip。

表格列顺序建议：

1. 业务编号或日期。
2. 人员信息。
3. 业务信息。
4. 状态标签。
5. 时间。
6. 操作。

### 4.3 表单

- 表单 label 宽度：100px 或 120px。
- 大表单按卡片分区。
- 必填项使用 rules 控制，不只在 label 中标星。
- textarea 显示字数限制。

### 4.4 弹窗与抽屉

| 场景 | 组件 | 说明 |
|:--:|:--:|:--|
| 简单新增/编辑 | `n-modal` | 用户、时间段、咨询室 |
| 查看复杂详情 | `n-drawer` | 预约审核详情、初访登记详情 |
| 危险操作确认 | `n-dialog` | 驳回、取消、删除、释放 |
| 表单较长 | 独立页面 | 首访登记、结案报告 |

### 4.5 标签

状态标签统一使用 `StatusTag`，风险标签统一使用 `RiskTag`。页面中禁止直接复制多套状态颜色逻辑。

## 5. 典型页面视觉设计

### 5.1 登录页

- 背景：浅蓝绿渐变。
- 左侧：项目名称和功能说明。
- 右侧：白色圆角登录卡片。
- 登录卡片宽度：360-420px。
- 按钮：全宽主色按钮。

### 5.2 管理员审核页

- 顶部指标卡：待审核、高风险、今日预约、排队人数。
- 下方搜索区。
- 表格高风险行浅红背景。
- 右侧抽屉展示预约详情。

### 5.3 学生预约页

- 使用步骤感：
  1. 填写登记表。
  2. 签署同意书。
  3. 选择预约时间。
- 时间段使用卡片，不建议只用下拉框。
- 不可预约时间段置灰并展示原因。

### 5.4 统计看板

- 顶部 5-6 个指标卡。
- 图表区域 2 列布局。
- 图表卡片高度建议 320-380px。
- 无数据时显示空状态，不显示空白图。

## 6. 交互反馈规范

### 6.1 成功反馈

使用 `message.success`，文案明确：

- “预约申请已提交，请等待管理员审核。”
- “初访结果已保存。”
- “正式咨询安排已生成。”
- “结案报告已提交。”

### 6.2 失败反馈

展示后端 message，尤其是业务冲突：

- “该咨询师在该时间段已有咨询安排。”
- “预约成功后撤销需至少提前一天。”
- “当前学生已有待审核预约，不能重复提交。”

### 6.3 二次确认文案

驳回预约：

> 确定驳回该预约申请吗？驳回后学生将看到驳回原因。

取消咨询安排：

> 确定取消该次咨询安排吗？取消后该时间段将释放。

提交结案报告：

> 确定提交结案报告吗？提交后管理员可以查看并下载。

## 7. 响应式要求

本项目以 PC 端演示为主，但页面应在常见笔记本屏幕下正常显示。

要求：

1. 1366px 宽度下无横向页面滚动。
2. 表格过宽时仅表格内部横向滚动。
3. 侧边栏在小屏下可折叠。
4. 图表宽度自适应卡片。
5. 表单在小屏下允许单列布局。

---

## 8. 可访问性与隐私提示

1. 涉及心理风险的内容不要在学生端使用刺激性文案。
2. 高风险标签主要显示给管理员、初访员、助理等工作人员。
3. 学生个人信息在表格中不展示过多敏感字段。
4. 通知、报告、咨询记录页面应避免大段隐私内容直接暴露在列表中，列表只显示摘要，详情需点击查看。

## 9. Naive UI 主题覆盖建议

可在 `src/styles/theme.ts` 或 `App.vue` 中配置：

```ts
export const themeOverrides = {
  common: {
    primaryColor: '#18A058',
    primaryColorHover: '#36AD6A',
    primaryColorPressed: '#0C7A43',
    borderRadius: '10px'
  },
  Card: {
    borderRadius: '14px'
  },
  Button: {
    borderRadiusMedium: '8px'
  }
}
```
