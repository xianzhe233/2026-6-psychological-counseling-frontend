# 前端实施计划

> 本计划用于指导前端从空仓库落地到可演示系统。每个阶段都应尽量形成可运行、可截图、可提交的小成果。

---

## 1. 开发前准备

### 1.1 初始化工程

建议命令：

```bash
npm create vite@latest . -- --template vue-ts
npm install
npm install vue-router naive-ui @vicons/ionicons5 axios echarts dayjs
```

如团队不熟悉 TypeScript，可使用 Vue JavaScript 模板，但仍建议保留 `src/types/` 文档化字段结构。

### 1.2 基础配置

需要完成：

1. 配置路径别名 `@ -> src`。
2. 配置环境变量 `VITE_API_BASE_URL`。
3. 配置 Vite 代理，解决开发环境跨域。
4. 引入全局样式。
5. 配置 Naive UI Provider。

验收标准：

- `npm run dev` 可启动。
- 浏览器访问首页无报错。
- Naive UI 按钮、卡片能正常显示。

---

## 2. 阶段一：基础框架

### 2.1 路由与布局

任务：

- 创建 `router/index.ts` 和 `router/routes.ts`。
- 创建 `AuthLayout.vue`、`MainLayout.vue`。
- 创建 `AppSidebar.vue`、`AppHeader.vue`。
- 创建 `/login`、`/dashboard`、`/403`、`/404` 页面。

验收标准：

- 登录页独立布局。
- 登录后页面有侧边栏、顶栏和内容区。
- 路由懒加载。
- 无权限路由可跳转 403。

### 2.2 API 基础封装

任务：

- 创建 `src/api/http.ts`。
- 配置请求和响应拦截器。
- 处理 401、403、409、500。
- 创建 `src/api/auth.ts`。
- 创建 `src/stores/auth.ts`。

验收标准：

- 页面中不直接使用 axios。
- 登录状态刷新后能通过 `/auth/current` 恢复。
- 未登录访问业务页面跳转登录页。

---

## 3. 阶段二：学生端主流程

### 3.1 首访登记表

任务：

- 创建 `FirstVisitFormView.vue`。
- 创建 `StudentBasicForm.vue`、`RiskQuestionForm.vue`。
- 实现表单校验。
- 调用保存接口。

验收标准：

- 必填项校验有效。
- 风险问题可填写。
- 提交成功后跳转知情同意页。

### 3.2 知情同意书

任务：

- 创建 `ConsentView.vue`。
- 展示同意书文本。
- 勾选后允许签署。

验收标准：

- 未勾选不能提交。
- 签署成功跳转预约页。

### 3.3 初访预约

任务：

- 创建 `AppointmentCreateView.vue`。
- 创建 `AppointmentSlotPicker.vue`。
- 根据日期加载可预约时段。
- 提交预约申请。

验收标准：

- 不可用时段置灰。
- 剩余容量展示正确。
- 提交成功后进入我的预约。

### 3.4 我的预约与通知

任务：

- 创建 `MyAppointmentsView.vue`。
- 创建 `MyNotificationsView.vue`。
- 实现预约撤销。

验收标准：

- 状态标签展示正确。
- 驳回原因、审核备注可查看。
- 可撤销时才显示撤销操作。

## 4. 阶段三：管理员端

### 4.1 基础信息管理

任务：

- 用户管理。
- 工作人员管理。
- 咨询室管理。
- 时间段管理。

验收标准：

- 每个页面具备查询、新增、编辑、启用/停用。
- 表单校验有效。
- 分页正常。

### 4.2 值班管理

任务：

- 值班列表。
- 新增/编辑值班。
- 批量排班。
- 冲突提示。

验收标准：

- 同一工作人员同一时间冲突时有明确提示。
- 可按日期和人员筛选。

### 4.3 初访预约审核

任务：

- 审核列表。
- 风险标签和高风险行样式。
- 详情抽屉。
- 通过、驳回、改约、优先。

验收标准：

- HIGH/URGENT 风险明显。
- 通过后状态刷新。
- 驳回必须填写原因。

---

## 5. 阶段四：初访员与心理助理端

### 5.1 初访员端

任务：

- 我的初访任务列表。
- 初访结果录入页面。

验收标准：

- 只能看到自己的任务。
- 结论为转介时后续建议必填。
- 结论为安排咨询后进入队列。

### 5.2 心理助理端

任务：

- 咨询队列列表。
- 咨询安排页面。
- 咨询时段预览。
- 冲突展示。

验收标准：

- 队列按优先级展示。
- 安排咨询时能预览当前选择的日期、时间、咨询师和咨询室。
- 冲突时不提交，并展示原因。

任务：

- 统计看板页面。

- 通知日志页面。

## 8. 前端提交建议

每完成一个可独立运行的小功能提交一次，例如：

1. `初始化 Vue3 前端工程`
2. `完成登录页与主布局`
3. `完成学生首访登记页面`
4. `完成学生预约页面`
5. `完成管理员预约审核页面`
6. `完成咨询安排页面`
7. `完成统计看板页面`

提交前检查：

```bash
npm run build
```

如配置了 lint，则同时运行 lint。

---

## 9. 前端演示数据依赖

前端演示至少需要后端提供：

1. 五类角色账号。
2. 初访员、咨询师、咨询室、时间段和值班数据。
3. 学生首访登记和预约数据。
4. 高风险预约数据。
5. 咨询队列和咨询安排数据。
6. 咨询记录和结案报告数据。
7. 统计看板数据。

若后端尚未完成，可先在页面中使用 mock 数据搭建静态效果，但最终必须切换到真实接口。
